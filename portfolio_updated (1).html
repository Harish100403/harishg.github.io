3-Phase Phase Sequence Detection & Protection Relay

Lead Firmware & HW Developer · PIC18/16 Bare-Metal C

Standard software HAL libraries introduce unpredictable delays, making reliable high-speed phase monitoring and zero-crossing detection difficult in high-power industrial stabilizer environments.

Bare-Metal Solution

  • Designed a dedicated fault detection system built entirely on register-level programming for PIC16/PIC18 microcontrollers — no HAL overhead.
  • Engineered a high-speed zero-crossing detection algorithm using direct bit-manipulation of GPIO and Timer Interrupts, achieving sub-1 ms response times.
  • Implemented precise phase-loss and phase-reversal protection logic triggering physical relays via low-level GPIO control instantly on fault recognition.
  • Optimised flash memory usage — entire control algorithm fits within the MCU's minimal footprint while maintaining maximum stability.
Result: Production-ready control card providing 100% reliable real-time protection against critical power faults in industrial stabilizers.

Automated Serial Number Programming over CAN Bus

System Architect & Firmware Developer · STM32 Bare-Metal & Python

Manual provisioning of unique serial numbers onto STM32 production units was slow, error-prone, and bottlenecked the manufacturing line.

Automation Solution

  • Architected a full production automation ecosystem linking a PC-based Python controller to the STM32 target hardware over CAN Bus.
  • Engineered high-efficiency STM32 firmware establishing a robust bare-metal CAN Bus stack configured at the register level for precise timing and baud rates (500 kbps).
  • Developed the custom Python "Production Host" application that generates, databases, and transmits unique serial IDs over CAN.
  • Created the STM32 flash programming routine accepting CAN data and safely writing IDs to designated internal flash pages — no JTAG/SWD required.
Result: Reduced provisioning from several minutes to under 10 seconds — zero human entry errors, significantly higher factory output.

3-Phase Servo Stabilizer Control Card with Data Logging

Firmware Engineer & PCB/Schematic Designer · PIC Bare-Metal

Traditional servo-stabilizers failed reactively but lacked diagnostics or logging for preventive maintenance or understanding prior power failures.

I2C / EEPROM Solution

  • Developed register-level PIC MCU firmware managing stabilization control loops for all three phases simultaneously.
  • Engineered and optimised a raw I2C communication stack interfacing a 24LC series EEPROM and DS1307 RTC — no library calls, maximum speed.
  • Implemented a fault-logging routine capturing timestamps (RTC) and fault codes instantly, committed to a ring-buffer on EEPROM via low-level I2C writes.
  • Designed a Python/UART PC utility to retrieve and decode fault history logs, transforming raw binary data into a readable failure report for technicians.
Result: Transformed reactive stabilizers into smart diagnostic systems — faster repairs and extended equipment lifespan across the fleet.
